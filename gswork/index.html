Security mechanisms in information systems

List of mechanisms

  1. Encipherment

    This is hiding and sealing data which allows it to be more confidential. This is achieved by turning data into a numerical form or applying mathematical calculations and algorithms creating unreadable information. The techniques known to do this are Encipherment and Cryptography. Typically, advanced encryption depends on the algorithm used for Encipherment.

  2. Access control

    This mechanism is in place to make sure that data cannot be accessed by people that should not have access. This is achieved by applying passwords, using pins or using firewalls so that hackers cannot breach the system. For example, if someone wanted to access private information they would have to get authorisation through the company by accessing passwords only known to them.

  3. Message Digest

    A fixed size numeric representation of the contents of a message, computed by a hash function. A message digest can be encrypted, forming a digital signature. Hash function must meet two criteria: The hash function must be one way, it must not be possible to reverse the function. Also, it must be computationally infeasible to find two messages that hash to the same digest. A message digest created using a secret symmetric key is known as a Message Authentication Code (MAC), because it can provide assurance that the message has not been modified.

  4. Data integrity

    Data integrity is important as it needs to be consistently verified. This is a security mechanism because if the data remains consistent it means that the data is reliable and not coming from a malicious source. This is achieved by sending information known to both sending and receiving parties and checked before and after the data has been received.

  5. Notarisation

    This security mechanism Is used act as a mediator between the sender and receiver so that information cannot be conflicted. This is typically done by a trusted third party and the mediator is requested to keep record of all requests that are made by the made sender.

  6. Authorisation exchange

    This security mechanism is in place to make sure that the correct parties are allowed to access information but only if the identity is known in communication. This achieved at the TCP/IP layer where a two-way handshake mechanism is in place.
